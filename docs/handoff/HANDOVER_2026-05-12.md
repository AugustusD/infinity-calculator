## 10. Domain crash course (minimum to be useful)

### Glass railing parts

- **Post:** vertical aluminum piece with a channel down its length. Glass slides into the channel. Topless (no top rail across the glass).
- **Glass insert / vinyl:** the rubber-ish strip that lines the channel. Holds the glass snugly. Sold by the 10ft or 12ft stick.
- **Setting block:** small vinyl piece at the bottom of the channel that supports the glass. Sold as 10ft stick, 1 1/2" pre-cut piece, or (in 2021) per foot.
- **Glass wedge:** 3" piece at the top of the channel that locks the glass in place. Vinyl.
- **Base plate:** the metal flange at the bottom of the post that attaches to the structure. 5" tall for fascia, integrated into the post for surface.
- **Wall track:** wall-mounted U-channel that terminates a glass run against a wall. Treated like a half-post.
- **2.5" end post (EP25):** smaller-profile post used at the very end of a run. Different SKU than the standard 2.5" Infinity post.

### Mount types

- **Surface mount:** post sits on top of the deck. Base plate at deck level. The "standard" mount.
- **Fascia mount:** post hangs off the side of the deck. Base plate attaches to the vertical fascia BELOW the deck. The post extends below the deck. This is where most bugs were found this session.

### Key dimensions

- **`railHeight`:** customer-facing dimension. 36" or 42" (nominal). React adds 1/8" gasket convention so internally it's 36.125 or 42.125. Excel uses the bare 36/42.
- **`topGlassReveal`:** how much glass sticks up above the top of the post. Standard = 2.125".
- **`bottomGlassGap`:** distance from the deck to the bottom of the glass. Standard = 2".
- **`distToDeck` (fascia only):** distance from the top of the base plate (below deck) DOWN to the deck. Standard = 3".
- **`postHeightAboveDeck` = `railHeight - topReveal`:** the visible portion of the post above the deck. ~40" standard.
- **`physicalPostLength` (fascia) = `postHeightAboveDeck + distToDeck + BASE_PLATE_HEIGHT(5)`:** the full post including the below-deck portion. ~48" standard.

### Setting block geometry

- **Surface:** sits on top of base plate. Height = `bottomGap - 0.5"` (subtracting the 0.5" base plate thickness already at deck level).
- **Fascia (post side):** fills from top of base plate (3" below deck) up to bottom of glass (2" above deck). Height = `bottomGap + distToDeck`. **Max 5 1/8" — if exceeded, switch to 3" wedge + balance.** (Rule 3 in section 9 above.)
- **Fascia (wall track side):** wall track is at deck level. Setting block height = `bottomGap - 0.5"` same as surface.

### Pricing math

- **Posts:** linear interpolation between two anchor heights (e.g., 34" and 40" for surface; 32" / 42" / 48" for fascia). React uses `surfacePostPrice(height)` and `fasciaPostPrice(height)` helpers.
- **Paint cost:** baked into the unit price; the separate "Paint Cost" column is informational only (not added to the line total or subtotal). Surface and fascia have DIFFERENT paint formulas (different anchors, fascia has × 1.1). Bug #15 fixed this for fascia.
- **Discount:** flat percentage applied to most lines. Fasteners are NET (no discount) — exception coded explicitly.

### The 1/8" gasket convention

React: `actualRailHeight = railHeight + 0.125` (includes a 1/8" base plate gasket).
Excel: uses bare `railHeight`.

Result: every dimension in React is 0.125" higher than in Excel. The `compare.py` script tolerates 0.13" drift to account for this. React's convention is **more accurate per IAS install guides** (which call out the gasket explicitly).

### The cut algorithm

`courierCutSpec(postHeight, thickness)` picks a manufacturer-fixed cut size and cuts-per-stock-length based on post height tier:

- 12mm thickness, 144" stock:
  - postHeight ≥ 38" → 48" cuts, 3 per stock
  - postHeight ≥ 27" → 36" cuts, 4 per stock
  - else → 24" cuts, 6 per stock
- 13mm thickness, 120" stock:
  - postHeight ≥ 38" → 40" cuts, 3 per stock
  - postHeight ≥ 27" → 30" cuts, 4 per stock
  - else → 24" cuts, 5 per stock

This is a manufacturer-fixed table — don't tweak without confirming new tiers with IAS.
